======================
ext4 文件系统
======================

ext4 是 ext2/ext3 的继任者，也是多数 Linux 发行版的默认根文件系统。它在保持向后兼容的同时，引入了 extents、延迟分配、更大容量等特性。本节介绍 ext4 的磁盘布局、inode 结构和日志机制。

ext 家族简史
========================

.. list-table::
   :header-rows: 1
   :widths: 12 18 50

   * - 版本
     - 年代
     - 主要特性
   * - ext2
     - 1993
     - 经典 Unix FS 布局，无日志
   * - ext3
     - 2001
     - 增加 journal，在线扩容
   * - ext4
     - 2008
     - extents、延迟分配、更大文件/FS

ext4 挂载为 ext3 或 ext2 时，部分新特性不可用，但数据可读。

磁盘布局
========================

ext4 将分区划分为:strong:`块组` （block group），每个块组包含：

.. code-block:: text

   块组 N
   ┌────────────────┐
   │ 超级块副本      │  （部分块组有备份）
   ├────────────────┤
   │ 块组描述符      │
   ├────────────────┤
   │ 块位图          │  哪些数据块已用
   ├────────────────┤
   │ inode 位图      │  哪些 inode 已用
   ├────────────────┤
   │ inode 表        │  多个 inode 结构
   ├────────────────┤
   │ 数据块          │  文件内容和目录项
   └────────────────┘

:strong:`超级块` 描述整个文件系统：块大小、块总数、inode 总数、UUID 等。挂载时首先读取超级块验证魔数（``0xEF53``）。

inode 与 extents
========================

ext2/3 用:strong:`直接块 + 一级/二级/三级间接块` 指针定位数据，大文件需多次间接访问。ext4 引入:strong:`extents` ：连续块可用一个 extent 描述（起始块 + 长度），减少元数据、提高大文件效率。

inode 结构（简化）包含：mode、uid、gid、大小、时间戳、块指针或 extent 树。目录文件的"数据块"存放 ``{inode, name_len, file_type, name}`` 格式的目录项。

.. code-block:: bash

   sudo dumpe2fs /dev/sda2 | head -50   # 查看超级块和块组信息
   sudo debugfs -R "stat <inode>" /dev/sda2   # 调试 ext 文件系统

日志（Journal）
========================

ext4 支持三种日志模式（挂载选项 ``data=``）：

.. list-table::
   :header-rows: 1
   :widths: 15 45

   * - 模式
     - 行为
   * - ordered
     - 默认。元数据日志，数据先写盘再提交元数据
   * - writeback
     - 元数据日志，数据写入顺序不保证，最快但风险较高
   * - journal
     - 元数据与数据都写日志，最安全，最慢

日志是环形的专用区域或外部文件（``journal`` inode）。崩溃后 ``fsck`` 或自动重放使元数据一致。

延迟分配与多块分配
========================

:strong:`延迟分配` （delayed allocation）：写入 page cache 时不立即分配磁盘块，直到 flush 或内存压力时再分配。好处：

- 合并连续写入，减少碎片
- 短生命周期文件可能从未落盘，直接丢弃

:strong:`多块分配器` （mballoc）尝试为相关文件分配相邻块，提升顺序读写性能。
