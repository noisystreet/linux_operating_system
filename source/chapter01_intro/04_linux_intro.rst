======================
Linux 简介
======================

如果你正在一台 Linux 机器上阅读这篇教程，那你已经绕过了最开始的"如何安装 Linux"这一步。不过，大多数人第一次接触 Linux 时都有同样的困惑：这么多发行版（distribution）到底有什么区别？内核和发行版是什么关系？这一节来梳理清楚。

内核 vs 发行版
================

严格地说，"Linux" 这个词有两层含义：

**Linux 内核** （Linux kernel）
  由 Linus Torvalds 维护的操作系统内核。它是 Linux 生态的核心——处理进程调度、内存管理、文件系统、网络协议栈、设备驱动等等。内核本身的版本号由 Linus 团队统一管理，例如 6.6、6.12。

**Linux 发行版** （Linux distribution）
  在内核的基础上，加上 GNU 工具集（GCC、glibc、coreutils）、包管理器、图形界面、桌面环境等，形成一个 **可直接安装使用的完整操作系统**。发行版的版本号是各自管理的，和内核版本无关。

举个例子，你在某台服务器上运行 Ubuntu 24.04，它的内核版本可能是 6.8。Ubuntu 24.04 是发行版号，6.8 是内核号，两者没有必然的对应关系。

主流发行版概览
================

Linux 发行版数量惊人（DistroWatch 上记录着数百个活跃项目），但绝大多数可以归为几个家族：

Debian 家族
-----------

- **Debian** ：以稳定性著称的社区发行版。软件包数量庞大（超过 6 万个），更新节奏保守。很多其他发行版基于 Debian。
- **Ubuntu** ：基于 Debian，更注重桌面体验和易用性。每两年发布一个 LTS（长期支持）版本。它是开发者中最流行的 Linux 之一。
- **Linux Mint** ：基于 Ubuntu，桌面体验更接近 Windows，适合从 Windows 迁移的用户。

Red Hat 家族
------------

- **Red Hat Enterprise Linux（RHEL）** ：企业级商用发行版，提供 10 年支持周期。被大量企业部署在服务器上。
- **Fedora** ：RHEL 的上游社区版本。新技术先进入 Fedora，稳定后进入 RHEL。适合愿意尝鲜的开发者。
- **CentOS** ：曾经的 RHEL 免费克隆版，现在转为 CentOS Stream——RHEL 的上游开发分支。

Arch 家族
---------

- **Arch Linux** ：滚动更新（rolling release），没有大版本的概念。用户自己动手搭建系统，适合深度定制。它的包管理器 `pacman` 和社区仓库 AUR 是其特色。
- **Manjaro** ：基于 Arch，但提供了图形安装器和更保守的更新策略，降低了 Arch 的上手门槛。

独立发行版
----------

- **openSUSE** ：分为 Leap（稳定）和 Tumbleweed（滚动更新），在桌面和运维社区中口碑很好。
- **Slackware** ：最古老的仍在活跃维护的 Linux 发行版之一，以简洁和设计一致性著称。

如何选择？如果你是初学者，从 Ubuntu 或 Fedora 入手会比较顺利。当你对系统有了更深的了解后，可以尝试 Arch 来真正"亲手搭建"一个 Linux 系统。

Linux 版本号的含义
====================

内核版本号的格式通常是 ``6.12.7``，其中：

- **6** ：主版本号（major），重大架构变化时增加
- **12** ：次版本号（minor），包含新特性——有趣的是，Linux 在 2.6 这个版本"卡"了将近 8 年（2003-2011），期间通过扩展的 `-r` 后缀发布新功能
- **7** ：补丁号（patch），bug 修复和安全更新

查看当前系统的内核版本：

.. code-block:: bash

   uname -r

查看发行版信息：

.. code-block:: bash

   cat /etc/os-release

GNU 与 Linux 的关系
====================

.. code-block:: text

   GNU 是"GNU's Not Unix"的递归缩写。

GNU 项目（前面历史一节提到过）提供了用户空间所需的几乎所有工具——编译器、库、shell、核心命令行工具。Linux 内核 + GNU 工具 = 一个完整的操作系统。这也是为什么技术圈里有人坚持称它为"GNU/Linux"而不是简单的"Linux"。

不过在日常使用中，人们通常简称"Linux"。重要的不是名称，而是理解这个生态的构成：内核来自 Linux 社区，用户空间工具来自 GNU 和开源社区，包管理系统来自发行版维护者。

Linux 的适用场景
=================

Linux 几乎无处不在。这里列几个和你日常会有交集的场景：

- **Web 服务器** ：全球超过 90% 的网站的服务器运行 Linux
- **云基础设施** ：AWS、Google Cloud、阿里云的核心实例都运行 Linux
- **Android 设备** ：Android 基于 Linux 内核——你的手机也运行着 Linux
- **嵌入式设备** ：路由器、智能电视、汽车信息娱乐系统
- **超级计算机** ：TOP500 排名中所有超算都运行 Linux
- **桌面开发** ：对于服务端开发者来说，Linux 是首选的开发环境

准备好进入动手环节了吗？下一节我们用几条命令，看看你面前这个 Linux 系统的具体面貌。

查看内核配置
==========================

运行中的内核编译选项可通过以下方式查看（若发行版启用了该功能）：

.. code-block:: bash

   # 部分发行版将配置压缩存放在 /proc
   zcat /proc/config.gz 2>/dev/null | less

   # 或查看 /boot 中的配置文件
   ls /boot/config-$(uname -r)

``CONFIG_*`` 选项决定内核是否包含某驱动、安全特性或调试功能。自行编译内核时，通常以当前配置为起点（见 :doc:`/appendix/a2_build_kernel`）。

主流发行版概况
==========================

.. list-table::
   :header-rows: 1
   :widths: 25 75

   * - 发行版
     - 特点与场景
   * - Debian / Ubuntu
     - 社区广泛，服务器与桌面均常见；Ubuntu 在 WSL 与云镜像中占有率高
   * - RHEL / Fedora / CentOS Stream
     - 企业级支持，RHEL 在商用服务器市场占重要份额
   * - Arch Linux
     - 滚动更新，面向进阶用户
   * - Android
     - 基于 Linux 内核的移动系统，全球活跃设备数以十亿计

精确的安装量难以统计（开源系统可自由复制），但服务器、超级计算机与云基础设施领域，Linux 长期占据绝对多数。
